JaplScript is a "less than perfect" bridge from Java to AppleScript. It allows you to generate Java interfaces based on .sdef files, which contain "scripting dictionaries". These scripting dictionaries essentially describe the AppleScript interface of an application and can be viewed and exported using the macOS Script Editor.

The generated Java interfaces are filled with life using object invocation handlers, essentially allowing a program to interact with an AppleScript-enabled application, almost as if it had a Java interface.

To generate Java interfaces, use the provided Ant task com.tagtraum.japlscript.generation.Generator:

<taskdef name="japlscript" classname="com.tagtraum.japlscript.generation.Generator" classpathref="some.classpath"/>
<japlscript application="Music" sdef="src/main/resources/Music.sdef" out="generated-sources/main/java" packagePrefix="com.apple.music"/>

As you can see in the sample above, you may need to provide a .sdef scripting dictionary (in this case the Music.app), an output path (here: generated-sources/main/java), and a package prefix (here: com.apple.music).

Once you have generated the interfaces, you can get a reference to the scriptable application using the method com.tagtraum.japlscript.JaplScript.getApplication() with the generated application interface and the application name as arguments:

com.apple.music.Application app = com.apple.music.Application.getInstance();
